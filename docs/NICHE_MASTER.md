# Niche 総合要件定義書（マスタードキュメント）

## 1. 前提・コンセプト
**メディアコンセプト**: 「知らなかった面白いに出会う場所」

**目的**: 需要（ペイン）はあるが供給がない領域のSEOを獲得し、専用品を探している層へ届ける。広告マネーゲームで埋もれる良品を検索意図ベースで拾い上げ、企業の狂気や一点突破の面白さを読み物（マガジン）として成立させる。

**基本ルール**:
- 最優先は「ニッチかどうか」。ストーリーの捏造は厳禁。
- 事実（出典あり）と推測（「※公式に記載はないが〜」等の明示）を完全に分離する。
- アフィリエイトは記事タイプで分け、全記事には配置しない。

## 2. ペルソナ（執筆者）設定
**名前**: Shinta（慎太）

**背景**: 本業でBPO（業務委託・プロセス構築）を手掛け、数多くの業界の裏側や業務効率化に携わる。同時に別事業でクリエイティブ制作や越境EC（D2Cブランド支援等）を展開し、国内外の無数のプロダクトに触れている。

**思考回路**: 気合いや根性を嫌い、「人の工数（時間）を削るツール」を愛する合理主義者。

**Niche開設の動機**: 「本当に誰かの深い悩みを解決する素晴らしいプロダクトが、資本力（広告費）の差だけで消費者に届かず埋もれていく現状」に対する強烈なアンチテーゼ。

## 3. コンテンツ設計と採用ゲート
### 3.1 記事タイプ（2レーン）
- **収益記事（全体70%）**: Amazon/楽天/Yahooで買える専用品（原則アフィリエイトあり）。
- **発見記事（全体30%）**: クラファン・日本未上陸・公式直販のみの製品（原則アフィリエイトなし、末尾に「今すぐ買える妥協案」としてアフィリンクを添えるのは可）。

### 3.2 ニッチ判定（採用ゲート・必須3条件）
以下の3つが全て揃わない情報は「ゴミ」として破棄し、生成APIを回さない。
- ペインが具体（状況が言える／場面が浮かぶ）
- 解決が一点突破（用途が狭い／専用すぎる）
- 供給が薄い（代替手段が妥協案しかない）

## 4. 自動化システム設計
**目標運用**: 完全自動下書き生成 → 人は「1日10分の承認（公開可否の判断）」のみ。

**インフラ**: Vercel（フロント＋API） ＋ データベース（下書き・出典・承認ログ保存）。

**LLMモデル**: Claude Sonnet 4.6（推論力と表現力の高さ、AI臭の排除、低コスト）。

**入力必須データ（5項目）**:
1. 基本情報（商品名/メーカー/価格）
2. 解決するペイン
3. 開発背景・ストーリー（最重要）
4. コア機能・特徴
5. 画像URL

**アフィリエイト動的生成**:
LLMには「検索用キーワード（商品名や型番）」のみを出力させ、Vercel側で各モールの検索結果URL（アフィリエイトタグ付き）を動的に組み立てる。

## 5. 記事生成フォーマット（完全版）と例文
LLMに絶対遵守させる出力構造と、その完成形プレビュー。

### 【出力構造】
- 導入（Shinta体験＋名もなき面倒の言語化）
- H1タイトル（現象＋ペイン＋最短ルート）
- 商品紹介（リード文と結論）
- H2：ニッチの深淵（開発背景・企業の狂気）
- H2：残酷な比較表（専用品 vs 妥協案）
- H2：圧倒的な一点突破（選ばれる理由・3〜5の箇条書き）
- H2：注意点（買う前の地雷・切り捨て）
- H2：ニッチFAQ（2〜4つ）
- 締め（読後感：ワクワク／合理的な手放しの提案）
- 署名（Written by Shinta）

#### 圧倒的な一点突破（使ってわかる面白さ）
- 「自分の指」のように薄切り肉を1枚ずつ剥がせます。 先端がぴったりと密着するように特殊な加工が施されており、指先と同じ感覚でスッと剥がせます。
- ニンニクや魚の「匂い移り」から解放されます。 手に匂いが染み付く食材を扱うときも、このトングを介せば一切手が汚れません。
- 逆さに置けば、先端が浮いて衛生的です。 調理途中にキッチン台にポンと置いても、先端がテーブルに触れない絶妙なカーブに設計されています。
