# 『Niche』メディア概要書（完全版 v1.2｜2026-02-24）

## 1. 目的
需要（ペイン）はあるが供給がない領域のSEOを先取りで獲得し、niche業界のプラットフォーム（マガジン）を作る。ピンポイントな「専用品」を探している層へ届ける。広告のマネーゲームによって埋もれてしまった純粋な「課題解決ツール」を救い出し、世に認知させる。

## 2. 狙い
大手メディアが書かない（書けない）「現象＋ペイン」の検索クエリを総取りする。ただのモノ売り（アフィリエイトサイト）ではなく、企業の狂気や一点突破の面白さを読み物として成立させることで、「ここに来れば面白い道具が見つかる」というマガジンとしての指名検索・信頼を獲得する。

## 3. 戦略
- **ニッチ判定の徹底**: 「ペインが具体」「解決が一点突破」「代替が妥協案しかない」の3条件が揃わない商品は、どれだけストーリーがあっても紹介しない。
- **検索意図の逆算**: 一般名詞（例:「トング おすすめ」）ではなく、行動と不満（例:「生肉 切る 手を洗う 面倒」）から記事を設計する。

## 4. 収益構造
- アフィリエイト（Amazon / 楽天 / Yahoo）
- 掲載依頼（企業）
- 広告収益

## 5. ターゲット
- **コア層**: 特定の状況下で悩んでいる人。普段は検索しない事例だが、たまに検索しても満足する情報がネットに無い層。
- **検索流入層**: 「特定の深いペイン」を持ち、汎用品や手作業といった既存の解決策（妥協案）の限界にすでに気づいている人。

## 6. 執筆者ペルソナ（Shinta）
- **背景**: 本業でBPO（業務構築）、別事業で越境ECやクリエイティブ等を手掛ける。
- **思考回路**: 好奇心旺盛で色々なものが好き。仕事柄アンテナを常に張って新しい情報やニッチな情報を集めている。
- **トーン＆マナー（重要）**: 文章内で「BPO」「EC」などの単語や仕事の話は一切出さない。あくまで日常の中の気づきや、仕事全体をベースに対して話す。

## 7. 情報抽出元と狙い
自動化・ネタ探しのための「3つのレーン」。

1. **Amazon / 楽天のレビュー・Q&A**
   - 狙い: ペインの逆算と検索クエリの素材収集。低評価レビューから「〇〇 使いにくい」等の生の不満を3つ以上抽出し、それを解決する専用品を探す。→アフィリエイト可能性大
2. **クラファン（Makuake / Kickstarter等）**
   - 狙い: 未来のニッチ製品の発掘。供給が薄く、開発者ストーリー（なぜ作ったかという狂気）が濃いものを拾う。→アフィリエイト可能性低（まだ商品がない可能性が高い）
3. **Yahoo!知恵袋 / Reddit / PR TIMES**
   - 狙い: インサイトの補強。読者のニッチな状況の生々しい言い回しを拾い、導入文の共感度を上げる。

## 8. 記事の本文構成（固定フォーマット）
1. **導入**: アイスブレイク、時事情報や本文にすんなり入れる日常エピソード、失敗談、体験、気づき。
2. **H1タイトル**: 「現象＋ペイン＋最適解（マガジン調）」（※原則、商品名は出さない）。
3. **リード文**: 既存の妥協案の限界を指摘し、結論（専用品）を提示。
4. **H2 ニッチな背景**: なぜわざわざ作ったのか（企業のニッチなこだわり・開発背景）。
5. **H2 比較表**: 専用品 vs 代替案1 vs 代替案2（手間・確実性・限界の3軸）。
6. **H2 ニッチポイント**: 読者の工数がどう減るかを3〜5つの箇条書き。
7. **H2 注意点**: 購入前に確認してほしいこと（明確な切り捨て）。
8. **H2 FAQ**: 一問一答（2〜4個）。
9. **締め**: 優しく背中を押す読後感 ＋ 署名（Written by Shinta）。

### 8-1. タイトルルール（マガジン調：正式仕様）

#### 【NG（量産型SEO）】
- `商品名｜〜を一発で解決する専用品`
- `商品名｜おすすめ/徹底解説/完全版`
- `商品名とは`

#### 【OK（マガジン調：Niche標準）】
- **商品名を主役にしない**。主役は「名もなき面倒（ペイン）」
- “妥協していた自分へのツッコミ”や“作り手の狂気”で読ませる

**型A（マスターデータ準拠）**
- `（現象＋ペイン）。（作り手/背景）が作った「○○」という最適解`
  - 例: 生肉を触るたびに手を洗う地味なストレス。燕三条の老舗が作った「第2の指」という最適解

**型B（マガジン調：自分へのツッコミ）**
- `なぜ今まで（妥協）していたのか。（現象）を「ただ（解決）するだけ」に全振りした専用品`
  - 例: なぜ今まで濡れた床を放置していたのか。シンクの水はねを「ただ防ぐだけ」に全振りした専用品

**補足（SEO拾いの実装ルール）**
- H1では商品名を出さない代わりに、本文冒頭で必ず以下を入れる（テンプレ固定）
  - `今回の専用品：{商品名}（{ブランド/メーカー}）`
  - `▶︎（検索リンク生成用キーワード：{商品名}）`
- slugは原則、商品名ベースで管理する（重複回避・運用のため）

### 8-2. なぜこのタイトルなのか（SEO / AEOの背景）
- **商品名検索**は「すでに知っている人」しか来ない。ニッチ専用品ほど母数が小さく、初動の流入が伸びにくい。
- **ペイン（自然言語）検索**は「困っている人」が来る。AIの普及で検索が単語から会話文に寄るほど、検索は「商品名」より「状況・不満（現象＋ペイン）」に引っ張られる。
- AEO（AIが回答を組み立てる検索体験）では、ユーザーの質問が「〜が面倒」「〜が不快」「どうすればいい？」のような**状況起点**になりやすい。
- そのためNicheは、タイトルを「ペイン主語」にしておくことで、**検索の入口（SEO）**と**AI回答で引用される入口（AEO）**の両方に適合させる。
- さらに「商品名が主役の量産型タイトル」を避けることで、マガジンとしての信頼（指名・回遊）を取りにいける。

## 9. 絶対ルール（AI・運用共通）
- **捏造の禁止**: 事実と推測を分離する。公式にない推測を書く場合は「※推測だが」と明示する。
- **表現の制限**: 「〜かもしれません」「いかがでしたか？」等の逃げの表現、過度なテンション、AI特有の一般論は禁止。
- **画像**: AI生成は禁止。実在の元画像URLを引っ張ってくる。
- **アフィリエイトリンク**: URLの直接生成はせず、「検索用キーワード」のみを指定する。（収益最大化）
